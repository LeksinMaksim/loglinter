# LogLinter

`loglinter` - это статический анализатор кода для языка Go, предназначенный для проверки содержимого лог-сообщений.

### Реализованные правила

1. **Регистр сообщения:** Лог-сообщения должны начинаться со строчкой буквы.
2. **Язык:** Допускается использования только английского языка.
3. **Спецсимволы и эмодзи:** Запрещено использование эмодзи, некоторых спецсимволов и избыточной пунктуации. Разрешенные символы: буквы, цифры, пробелы, `.,:=-[]_`
4. **Безопасность:** Поиск потенциальной утечки чувствительных данных (пароли, API-ключи, токены, секреты) по ключевым словам в строковых литералах.

### Технические требования
- **Go:** 1.22 и выше
- **Поддерживаемые логгеры:** `log/slog`, `go.uber.org/zap` (включая `SuggaredLogger`)
- **Совместимость:** Может работать как самостоятельная утилита или как плагин для `golangci-lint`.

### Способы запуска
#### 1. Как самостоятельный продукт

Вы можете собрать и использовать линтер как отдельный бинарный файл.

**Сборка:**
```shell
go build -o loglinter ./cmd/loglinter/main.go
```

**Запуск:**
```shell
# Проверка конкретного файла
./loglinter ./path/to/your/file.go

# Проверка всего пакета
./loglinter ./...
```

#### 2. Интеграция в `golangci-lint`

**Сборка:**
Убедитесь, что у вас установлен `golangci-lint`. Выполните команду в корне проекта:
```shell
golangci-lint custom
```

Эта команда прочитает файл `.golangci.yml`, загрузит плагин как модуль и соберет новый исполняемый файл (по умолчанию он называется `custom-gcl`)

**Запуск:**
```shell
# Проверка конкретного файла
./custom-gcl run ./path/to/your/file.go

# Проверка всего пакета
./custom-gcl run ./...
```

_Для тестирования линтера был подготовлен файл `test.go`, находящийся в корне проекта. Попробуйте запустить линтер для проверки этого файла._

### Тестирование

Линтер покрыт тестами с использованием `analysistest`. Тестовые сценарии (правильные и неправильные) находятся в `pkg/analyzer/testdata/src/a`.

**Запуск всех тестов:**
```shell
go test ./pkg/analyzer/...
```

### Ограничения

Статический анализатор работает на этапе компиляции, поэтому он проверяем только те сообщения, содержания которых можно определить без запуска кода. Линтер корректно обрабатывает:
1. **Строковые литералы**: `log.Info("starting server")`
2. **Простую конкатенацию (строка + переменная):** `log.Info("user password: " + password)` - проверяется только строковая часть литерала.
